# DocDiffAgent 사용 가이드 (v2.0 - 심층 분석 에디션)

`DocDiffAgent`는 두 버전의 기술 문서를 대조하여, 단순 요약을 넘어 **코드 레벨의 차이점과 기술 명세 변화를 정밀하게 분석**합니다. 

## 1. 주요 특징
- **전수 분석(Full-Scan)**: 매핑된 모든 섹션과 신규 추가된 섹션을 개별적으로 분석합니다.
- **고해상도 리포트**: 기존보다 3배 이상의 정보 밀도를 목표로 하며, `maxTokens(65536)` 설정을 통해 방대한 양의 기술 대조표를 생성합니다.
- **코드 중심 분석**: 어노테이션 변화, 메서드 시그니처, 설정 프로퍼티의 변경 사항을 중점적으로 추출합니다.

## 2. CLI 명령어 사용법

```bash
doc-diff --source-ver [소스버전] --source-path [소스경로] --target-ver [타겟버전] --target-path [타겟경로] [옵션]
```

### 실행 예시
```bash
doc-diff --source-ver 0.3.4 --source-path 0.3.4.md --target-ver 4.0.0 --target-path 4.0.0.md
```

### 옵션 상세
- `--source-ver`: 기준 버전 (예: `0.3.4`)
- `--source-path`: 기준 문서 파일 경로
- `--target-ver`: 비교 대상 버전 (예: `4.0.0`)
- `--target-path`: 비교 대상 문서 파일 경로
- `-p`: LLM에 전달되는 기술 분석 프롬프트를 콘솔에 출력
- `-r`: LLM으로부터 수신한 상세 JSON 응답 데이터를 콘솔에 출력

## 3. 분석 보고서 구성
생성된 보고서는 `output/DocDiffAgent/` 하위 워크스페이스에 `diff-report.md`로 자동 저장되며 다음 내용을 포함합니다:

1. **아키텍처 및 버전 개요**: 릴리스의 핵심 기술 방향성.
2. **기존 기능 변경 (Deep-Dive)**: 섹션별 상세 API/코드 명세 대조.
3. **신규 도입 기능**: AWS Bedrock, Guardrails 등 신규 모듈의 기술적 가치 분석.
4. **마이그레이션 체크리스트**: 0.3.4 사용자를 위한 5가지 이상의 핵심 체크 포인트.
5. **호환성 리포트**: 하위 호환성 파괴 항목(Breaking Changes) 명시.

## 4. 작동 아키텍처
1. **Structural Mapping**: `grep`을 통해 문서 골격을 파악하고 두 버전 간 섹션을 매핑합니다.
2. **Content Analysis Loop**: 매핑된 섹션과 신규 섹션의 본문을 개별적으로 읽어 LLM이 기술 감사(Technical Audit)를 수행합니다.
3. **High-Density Synthesis**: 수집된 명세 데이터를 종합하여 고밀도 백서를 합성합니다. (최대 65,536 토큰 지원)
